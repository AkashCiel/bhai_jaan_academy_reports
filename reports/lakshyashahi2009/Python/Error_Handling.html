Key Concepts of Error Handling

What is Error Handling?

Error handling involves anticipating, detecting, and responding to errors that occur during program execution. In Python, errors can arise from various sources, including invalid inputs, hardware failures, or logical errors within the code. By handling these errors effectively, developers can prevent crashes, provide meaningful feedback to users, and ensure that programs can continue running in a controlled manner.

Types of Errors in Python

Errors in Python can be broadly categorized into three main types:

    • Syntax Errors:
    • Occur when the code is not written according to Pythonâ€™s rules.
    • Example: Missing colons, incorrect indentation, or misspelled keywords.
    • Detection: Caught by the Python interpreter before execution.

python

# Syntax Error Example

print("Hello World" # Missing closing parenthesis

    • Runtime Errors:
    • Happen during program execution, often due to invalid operations.
    • Example: Division by zero, accessing an out-of-bounds index in a list.
    • Detection: Not detected until the program is running.

python

# Runtime Error Example

x = 10

y = 0

print(x / y) # Division by zero

    • Logical Errors:
    • Occur when the program runs without crashing, but produces incorrect results.
    • Example: Miscalculating a sum or failing to iterate through a loop correctly.
    • Detection: Often the hardest to identify, as the code runs without errors.

python

# Logical Error Example

total = 0

for i in range(5):

total += i # Should be total += i + 1 to count from 1 to 5

Exception Handling

In Python, errors are managed using a mechanism called exception handling. An exception is an event that disrupts the normal flow of a program. Python provides specific keywords to handle exceptions effectively:

    • try: This block of code is executed and is monitored for exceptions.
    • except: This block of code runs if an exception occurs in the try block.
    • else: This block runs if no exceptions occurred in the try block.
    • finally: This block runs regardless of whether an exception occurred or not, often used for cleanup actions.

Basic Syntax of Exception Handling

Here is a basic structure demonstrating exception handling in Python:

python

try:

# Code that may cause an exception

result = 10 / 0

except ZeroDivisionError:

# Handling the specific exception

print("You cannot divide by zero!")

else:

# Runs if no exceptions were raised

print("The result is:", result)

finally:

# Cleanup code that always runs

print("Executing finally block.")

Raising Exceptions

Sometimes, you may want to raise an exception intentionally. The raise statement is used for this purpose. It allows you to create custom exceptions to enforce certain conditions in your code.

```python

def check_age(age):

if age < 18:

raise ValueError("Age must be 18 or older.")

return "Access granted."

try:

print(check_age(16))

except ValueError as ve:

print(ve)

```

Creating Custom Exceptions

You can create your own custom exception classes by inheriting from the built-in Exception class. This is useful when your application requires specific error handling that cannot be covered by standard exceptions.

```python

class MyCustomError(Exception):

pass

def demo_function():

raise MyCustomError("This is a custom exception.")

try:

demo_function()

except MyCustomError as e:

print(e)

```
